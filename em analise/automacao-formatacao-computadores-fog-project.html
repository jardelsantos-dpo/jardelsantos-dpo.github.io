Como automatizar a formatação com FOG Project (passo a passo)

Se você trabalha com TI em uma pequena ou média empresa, provavelmente conhece o pesadelo logístico que é preparar computadores novos ou recuperar máquinas com problemas. O processo manual — baixar a ISO do Windows ou Linux, criar um pen drive bootável, instalar o sistema, baixar drivers, instalar atualizações, configurar os aplicativos e as políticas de segurança — consome, em média, de 2 a 3 horas por equipamento.

Multiplique isso por 10 máquinas novas e você perdeu pelo menos metade de uma semana de trabalho produtivo.

Com a "aposentadoria" do MDT, uma das soluções para realizar deploy de sistemas operacionais via rede de forma simples e econômica (open source) é o FOG Project.

Neste guia, baseado nas melhores práticas, apresentamos como transformar um PC simples ou uma máquina virtual em um servidor de imagens capaz de preparar diversos equipamentos simultaneamente, garantindo que todas as máquinas da sua empresa tenham exatamente o mesmo padrão de segurança e configuração.

O que é o FOG Project e por que sua empresa precisa dele?

O FOG Project é uma solução de clonagem e gerenciamento de imagens de computadores baseada em Linux e código aberto (gratuito).

Benefícios Estratégicos:

  • 1. Padronização e Hardening do SO: Ao criar uma imagem base focada nas configurações do Sistema Operacional (Hardening), você garante que o Windows ou Linux já nasça com as políticas de segurança, GPOs locais e ajustes de rede corretos, eliminando o erro humano. Softwares de segurança que exigem atualizações frequentes (como EDR e Antivírus) são instalados automaticamente logo após a formatação (via scripts ou Snapins), garantindo que a máquina sempre receba a versão mais recente da proteção.
  • 2. Velocidade de Recuperação (Disaster Recovery): Se um computador for infectado por ransomware ou tiver falha no disco, você pode restaurar a imagem original limpa em menos de 15 minutos via rede.
  • 3. Economia: Como é uma ferrame3nta com licença GPL (gratuito) não há necessidade de gastos com licenças de softwares proprietários de clonagem.
  • 4. Gestão de Inventário: O FOG também coleta dados de hardware das máquinas, ajudando no inventário de ativos, mas como o foco da ferramenta não é esse, essa coleta é mínima.

Pré-requisitos e Preparação do Ambiente

Você não precisa de máquinas caras para aplicar esta solução: até um computador antigo ou uma Máquina Virtual (VM) já dão conta de gerenciar redes com até 100 equipamentos.

Requisitos recomendados para o servidor FOG:

  • Processador: Dual Core ou superior.
  • Memória RAM: 4GB (8GB recomendado).
  • Disco: O uso de SSD é altamente recomendado para garantir maior velocidade. A capacidade necessária dependerá da quantidade de imagens que serão armazenadas — 250GB é um bom ponto de partida. No entanto, considerando boas práticas de governança e gestão, o ideal é manter apenas a imagem base do sistema operacional. Quanto menos imagens forem mantidas, mais rápido e fácil será administrar e atualizar o sistema.
  • Sistema Operacional: Ubuntu Server 24.04 LTS ou versões posteriores, reconhecido pela estabilidade e amplo suporte, ou Debian 13 ou versões posteriores.
  • Rede: Conexão cabeada (Ethernet) Gigabit. Não use Wi-Fi para o servidor.

Nota: Para este artigo utilizaremos como referência o Ubuntu instalado em uma máquina virtual com acesso à internet.

Passo 1: Preparando o Sistema Operacional

Antes de instalar qualquer ferramenta complexa, a regra de ouro da segurança e estabilidade é atualizar o sistema base. Acesse o terminal do seu servidor Ubuntu e execute:

1. Torne-se superusuário (root) para facilitar a execução dos scripts:

sudo -i

2. Atualize os repositórios e os pacotes instalados:

apt-get update && apt-get upgrade -y

3. Instale o Git (necessário para baixar o FOG):

apt-get install git -y

Passo 2: Baixando e Instalando o FOG Project

Diferente de programas do Windows onde você baixa um .exe, no Linux baixamos o código fonte direto do repositório oficial. Isso garante que você tenha a versão mais recente e segura.

1. Navegue até a pasta de arquivos temporários:

cd /opt

2. Clone o repositório do FOG Project:

git clone https://github.com/FOGProject/fogproject.git

Este comando cria uma cópia completa dos arquivos de instalação na sua máquina.

3. Entre na pasta do instalador:

cd fogproject/bin

4. Inicie o script de instalação:

./installfog.sh

Passo 3: O Assistente de Instalação

Ao rodar o script, o FOG fará uma série de perguntas sobre como você deseja configurar o servidor. Abaixo, detalho as respostas ideais para uma instalação padrão em uma pequena ou média empresa.

  1. OS Selection: O instalador detectará automaticamente que é um Debian/Ubuntu. Pressione Enter.
  2. Installation Type (Normal vs Storage): Digite N para Normal. Explicação: "Normal" instala o servidor completo (interface web + armazenamento). "Storage" seria apenas para adicionar mais espaço em disco em outro servidor secundário.
  3. Interface de Rede: O sistema sugerirá a placa de rede padrão (ex: eth0 ou ens33). Verifique se é a correta e pressione Enter.
  4. Change Default Network Settings? Digite N (Não), a menos que você precise fixar um IP diferente do que a máquina já pegou. Recomendação: É vital que o servidor FOG tenha um IP Fixo.
  5. Setup Router Address for DHCP? Digite Y (Sim) e confirme o IP do seu roteador (Gateway).
  6. Setup DNS? Digite Y (Sim) e confirme o IP do DNS (geralmente o mesmo do roteador ou 8.8.8.8).
  7. Would you like to use the FOG server for DHCP service?
    Atenção Crítica: Se você já tem um servidor DHCP na rede (como um Windows Server ou o próprio roteador da internet), digite N. Dois servidores DHCP na mesma rede causam conflito e derrubam a internet. Se esta for uma rede isolada/laboratório apenas para formatação, digite Y. Para este guia, assumiremos que você já tem um DHCP (roteador) e digitaremos N.
  8. Internationalization Support? Digite N (Opcional).
  9. HTTPS Support? Digite N para facilitar a configuração inicial em rede interna.
  10. Confirm inputs? Revise os dados e digite Y.

O instalador começará a baixar e configurar todos os serviços (Apache, PHP, MySQL, TFTP, NFS). Isso pode levar alguns minutos.

O Passo "Pulo do Gato": Atualização do Banco de Dados
Quase no final da instalação, o terminal vai pausar e mostrar uma mensagem grande pedindo para você acessar uma URL via navegador. Não pressione Enter ainda!

  1. Abra o navegador no seu computador.
  2. Digite o endereço IP do servidor FOG conforme mostrado na tela (ex: http://192.168.1.50/fog/management).
  3. Você verá uma tela pedindo para instalar/atualizar o banco de dados. Clique no botão azul "Install/Update Now".
  4. Aguarde a barra de progresso finalizar.
  5. Volte ao terminal do servidor Linux e, agora sim, pressione Enter para finalizar a instalação.

O instalador fornecerá o usuário padrão (fog) e a senha (password).

Passo 4: Configurando a Rede para o Boot PXE (DHCP)

Se você optou por não usar o FOG como DHCP (o cenário mais comum em empresas), você precisa "ensinar" o seu roteador ou Windows Server a redirecionar computadores novos para o FOG. Você deve acessar as configurações do seu servidor DHCP atual e configurar duas opções:

  • Option 66 (Boot Server Host Name): Insira o IP do servidor FOG.
  • Option 67 (Bootfile Name):
    • Para computadores modernos (UEFI): digite ipxe.efi
    • Para computadores antigos (Legacy/BIOS): digite undionly.kpxe

Dica de Especialista: Se você tem um ambiente misto, configurar isso no roteador simples pode ser difícil. A melhor prática é configurar todas as máquinas novas para UEFI e usar ipxe.efi.

Passo 5: Criando sua Primeira Imagem (Captura)

Agora que o servidor está pronto, vamos criar a "Imagem Mestra".

  1. Prepare o Computador Modelo:
    • Pegue um computador, formate-o do zero com Windows 10/11 ou Linux.
    • Instale as atualizações do Windows/Linux e softwares "estáticos" ou grandes (como Pacote Office e Runtimes C++).
    • Dica de Segurança: Evite instalar o Antivírus/EDR ou ferramentas de VPN na imagem base, pois eles desatualizam rápido e podem gerar conflitos de ID. A melhor prática é usar os Snapins do FOG (ou scripts de logon) para instalar esses agentes silenciosamente no primeiro boot após a clonagem.
    • Importante (Windows): Antes de desligar, execute o Sysprep (C:\Windows\System32\Sysprep\sysprep.exe). Marque a opção "Generalize" e escolha "Shutdown". Isso remove identificadores únicos (GUIDs), garantindo que, ao clonar, as máquinas não tenham conflito de identidade na rede.
  2. Cadastre o Host no FOG:
    • Ligue esse computador e configure a ordem de Boot para iniciar pela Rede (PXE/Network).
    • Ao iniciar, você verá o menu do FOG. Escolha a opção "Quick Registration and Inventory".
    • O computador vai reiniciar. Desligue-o.
  3. Crie a Definição de Imagem na Web:
    • Acesse o painel web do FOG. Vá em Images > Create New Image.
    • Dê um nome (ex: "Win11_Padrao_RH_2026").
    • Em "Operating System", escolha Windows 10/11 e em "Image Type", escolha "Single Disk - Resizable".
  4. Associe e Capture:
    • Vá em Hosts, ache o computador registrado e associe a imagem criada.
    • Vá na aba Basic Tasks e clique em Capture. Agende a tarefa.
    • Ligue o computador modelo novamente (com boot pela rede) e a cópia para o servidor começará automaticamente.

Passo 6: Implantando a Imagem (Deploy)

A mágica acontece agora:

  1. Pegue os computadores novos e configure o boot pela rede.
  2. No menu do FOG, faça o "Quick Registration".
  3. No painel web, selecione os novos computadores em Hosts.
  4. Defina a imagem "Win11_Padrao_RH_2026" para eles.
  5. Clique em Task > Deploy.

Os computadores irão reiniciar e baixar o sistema pronto simultaneamente. Em minutos, você terá sua frota pronta para uso e idêntica ao original.

Segurança e LGPD: O que você precisa saber

O uso do FOG Project toca diretamente em pontos cruciais da Segurança da Informação e LGPD:

  1. Descomissionamento Seguro (Wipe): O FOG possui funções de "Wipe" (limpeza de disco). Antes de doar ou descartar computadores antigos da empresa, use o FOG para sobrescrever os dados, garantindo que informações de clientes ou segredos de negócio não vazem em lixões eletrônicos.
  2. Segurança do Servidor: Como o FOG armazena cópias completas dos sistemas da sua empresa, ele é um ativo crítico. Altere a senha padrão do usuário fog imediatamente e não exponha a interface web para a internet.

Conclusão

Implementar o FOG Project é um divisor de águas na maturidade de TI de uma pequena empresa. Você deixa de ser um "formatador de máquinas" e passa a ser um gestor de infraestrutura. O tempo ganho na automação deve ser reinvestido em tarefas mais nobres, como a análise de vulnerabilidades, treinamento de usuários e melhoria dos processos de segurança. A automação não custa caro; ela custa apenas a vontade de aprender e implementar.
